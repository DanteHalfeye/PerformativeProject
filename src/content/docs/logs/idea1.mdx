## PROJECT GOAL

Create a DOOM-style interactive game where:

UNITY = GAMEPLAY / SIMULATION ENGINE
TOUCHDESIGNER = CUSTOM REAL-TIME VISUAL RENDERER

## UNITY RESPONSIBILITIES

Unity handles:

* Player movement
* FPS controls
* Weapons
* Enemy AI
* Physics
* Collision
* Level logic
* Game state
* Audio playback
* Object spawning

## TOUCHDESIGNER RESPONSIBILITIES

TouchDesigner handles:

* Particle rendering
* Sprite visualization
* Outline effects
* Procedural graphics
* Audio-reactive visuals
* GLSL shaders
* Lighting effects
* Post-processing

# IMPORTANT IDEA

Unity does not send the final image.

Unity sends information about the world.

Example data:

Camera:

* Position
* Rotation
* Field of view

Player:

* Position
* Rotation
* State

Enemies:

* ID
* Position
* Rotation
* Animation frame
* Health

Weapons:

* Bullet positions
* Impacts
* Explosions

Audio:

* Beat intensity
* Bass
* Frequency energy

TouchDesigner interprets this information and creates the visuals.

# FIVE WEEK DEVELOPMENT PLAN

WEEK 1 — COMMUNICATION PROTOTYPE

Unity:

* Create FPS controller
* Add enemies
* Send positions through OSC/UDP

TouchDesigner:

* Receive data
* Display points
* Match camera movement

Result:

Unity objects appear inside TouchDesigner.

WEEK 2 — BASIC RENDERER

Build:

* Camera system
* Sprite rendering

Playable scene rendered through TouchDesigner.

WEEK 3 — VISUAL IDENTITY

Add:

* GLSL effects

Result:

The game develops its own visual style.

WEEK 4 — OPTIMIZATION

60 FPS.

WEEK 5 — PRESENTATION

# PERFORMANCE RULES

Avoid sending:

* Complete frames
* Static information every frame
* Unnecessary object data

Send:

* Object positions
* Rotations
* Animation states
* Events

Use:

* GPU instancing
* GLSL shaders
* Object pooling
* Culling

# FINAL PROJECT VISION

A DOOM-inspired FPS where:

* Unity controls gameplay.
* TouchDesigner controls the appearance.
* Sprites remain recognizable.
* Outlines become particles.
* Music changes the environment.
* Explosions create visual events.
* The renderer behaves like a live VJ system.

The goal is not to replace Unity.

The goal is to create a custom visual interpretation of a game world.
